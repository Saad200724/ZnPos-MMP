---
name: MongoDB numeric IDs
description: The shared Zod schemas (generated by orval) use id: number throughout. MongoDB uses ObjectId. We bridge this with a Counters collection.
---

All shared types in `shared/src/generated/api.ts` have `id: zod.number()`. Changing these would break the frontend.

**Why:** Orval-generated types are not manually edited, and the POS frontend expects numeric IDs for customers, products, transactions, etc. MongoDB's ObjectId is not compatible.

**How to apply:** Each Mongoose model stores a custom `id: Number` field (alongside MongoDB's `_id: ObjectId`). A `Counter` model in `server/db/models/Counter.ts` holds a `{ _id: collectionName, seq: Number }` document per collection. Call `nextId("products")` before `Product.create()` to get the next numeric ID. Queries use `Model.findOne({ id: numericId })` instead of `findById`.

Externalize `mongoose` and `connect-mongo` in `server/build.mjs` for production builds.
